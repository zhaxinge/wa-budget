# WA Budget Lens — Large File Safe Version

Your old browser flow can go blank with large Excel files because the page reads the whole workbook and converts the entire first sheet to JSON in the browser main thread.

This version avoids that problem:

1. Precompute `summary.json` once from the large Excel/CSV file.
2. Host or upload `summary.json`.
3. The app sends only VERIFIED_FACTS + compact summaries + up to 200 sample rows to Claude.

## Repo structure

```text
wa-budget/
├── index.html
├── summary.json          # generated by the Python script
├── package.json
├── api/
│   └── ask.js
└── tools/
    └── precompute_summary.py
```

## Generate summary.json

```bash
pip install pandas openpyxl
python tools/precompute_summary.py Vendor-Payments.xlsx summary.json
```

For CSV:

```bash
python tools/precompute_summary.py vendor_payments.csv summary.json
```

## Deploy

- GitHub Pages hosts `index.html` and `summary.json`
- Vercel hosts `api/ask.js`

In Vercel Environment Variables:

```text
ANTHROPIC_API_KEY = sk-ant-...
CLAUDE_MODEL = claude-3-5-haiku-20241022
```

## Why this fixes blank page

The browser no longer parses the large Excel file. It loads a much smaller JSON file instead.
